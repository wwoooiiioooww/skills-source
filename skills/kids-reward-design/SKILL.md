# kids-reward-design — 報酬経済・承認ゲート・子ども語彙の設計

Studyきち（小5・小3の姉妹＋保護者向け勉強モチベーションPWA）v1〜v11の実開発から抽出。技術構成は pwa-multiapp Skillに準拠（localStorage `studykichi_v1`、グローバル `S`、`P()`/`save()`/`render()`、モーダルは `openModal()` 上書き式）。

## 絶対原則

1. **ハマっている体験を壊さない。** 既存動線の変更は「追加」より一段慎重に。新機能は既存画面への「1行・1カード」の追加に収め、コア動線（出撃→タイマー→記録→承認→スロット）には触れない。
2. **ごほうびは親の承認ゲートを通る。** スロット/ボーナスの実体は必ず承認後に届く。承認前に確定させない。即時に出してよいのは達成感の演出（紙吹雪・バッジ）のみ。承認ゲートには必ず実効果（スロット解禁等）を紐づける——バッジ表示が変わるだけの「効果のない承認」は作らない。
3. **正直な自己申告は報酬に連動させない。** 集中度レビュー等を報酬に繋げた瞬間「正直さより得」が勝ち、データが腐る。
4. **報酬経済は搾取耐性を持つ。** 時間係数（短時間連打×0.3）、日次条件（累計25分）、二重付与防止（週オブジェクトの`granted`フラグ、記録の「編集」で再付与されないこと）、レア確率は控えめから。緩めるのは後からできるが、締めるのは反発を生む。
5. **ポイントには必ずシンク（使い道）を用意する。** 「貯めて何になるの?」問題を作らない。
6. **設定より自動判定。** 「土曜に立てた作戦は来週扱い」「日曜のバトルは先週扱い」のように文脈から自動で正しく振る舞う。設定項目は最後の手段。
7. **子どもの言葉と世界観。** ひらがな多め・小学生語彙。命名は世界観（きち/しゅつげき/ボスバトル/リベンジリスト/さくせん会議）に乗せる。機能名がワクワクするかは仕様の一部。レア度は%でなく「超レア🌈/レア/ときどき/よくでる」。親向け（パパ・ママモード内）は通常の日本語でOK。
8. **確率の下方修正は体験の整合を保って行う。** 子どもに見える場所で宣言せず、画面上の表記も同時に変更する。

## 実装チェックリスト

- [ ] 承認フロー: pending→親が承認→子の画面に出現。却下時の挙動。せっていのバッジ件数（セッション+ボーナス合算）
- [ ] draft状態管理: 再レンダリングで入力値を失わない（`oninput`でdraft同期・チップ切替のみ`render()`、draftはプロフィール切替・週替わりで無効化）
- [ ] モーダル上書き仕様: モーダル内から別モーダルを開く導線には「←もどる」を付ける
- [ ] 入力ゼロ設計: 間違えた問題の自由記入ではなく「正解<問題数の教科」の自動リベンジリスト化のような自動導出を優先
- [ ] 空作戦防止: 報酬目当ての空入力を防ぐ条件（全教科入力完了まで決定不可等）があるか
- [ ] 報酬の告知: 「承認すると🎁とどくよ」→承認→次回起動時にお祝いカード。承認を翌日のログイン動機に変える
- [ ] ルール説明は控えめリンク→モーダル。確率表の常時表示で情報過多にしない
- [ ] AIレポート構成: 🌟たっぷりほめる→🔍GAPはやさしく1つだけ→💡ヒント2つ→👨‍👩‍👧親に自慢するコーナー。順番と分量比が本体

# kids-edu-html — 子供向けAI教育カリキュラムHTML教材の制作

目的: どのAI（どのモデル・どのセッション）が実装してもトーンと品質が揃う状態で、全12回を貫く一本の縦糸（情報の物語: 宇宙→DNA→脳→AI→未来)を子供の記憶に定着させる。個々の回の完成度だけでなく、伏線と回収の一貫性が成果物の価値そのもの。

## 絶対原則

1. **予想先行（Predict-first）**: 必ず「答えを見せる前に予想させる」。驚き→種明かし→納得、の順を崩さない。
2. **断定しない設計**: 「AIに心はあるか」のような答えのない問いは、コンテンツ側が結論を出さない。正解を確定的に出さないことそのものが設計目標になる回もある。
3. **恐怖で終わらせない**: 危険性を扱う回は必ず「対処できる/知っていれば怖くない」着地を用意する（例: 暴走シミュレーターに「🛑ルールを足す」ボタンで安全に止まる体験を必ず入れる）。
4. **1画面1概念、縦糸は各回1本**: 前回の回収→今回の核心→次回への橋、を必ず明示する。
5. **年齢に応じた言葉の翻訳義務**: 専門用語は対象年齢の実感に合う言葉に言い換える（境界線→なわばり、正解率→せいかい◯/◯こ）。専門語は消さず**大人向けコラムに退避させて温存**する。
6. **入力させたら負け**: 操作はタップ/クリック/スライダー中心。テキスト入力を要求しない。
7. **共創であることを隠さない**: footerに「この教材はAIと一緒に作りました」を明記する。

## 実装チェックリスト

- [ ] CSS変数（背景色・パネル色・5色の時代色・3書体）を新規定義せず、基準HTMLからコピーして再利用したか
- [ ] 署名要素（その回だけのcanvas/HTMLシミュレーター）が1回1つあるか。使い回していないか（配色や仕組みの引用で仲間感を出すのは可）
- [ ] prefers-reduced-motion対応。動きが止まっても伝わる静止表示があるか
- [ ] `<script>`をすべて抽出し `new Function()` 等でパースエラーがないか機械確認したか（目視だけで済ませない）
- [ ] 冒頭に前回の回収、末尾に次回への橋があり、「情報の物語」のどの地点かを一文で示したか
- [ ] 子供向け本文/進行メモ（保護者向け）/おとなコラムを `<details>` クラスで分離したか
- [ ] 時事ネタ・センシティブ情報は要ファクトチェック注記付きで講師向けコラムに隔離したか（本文に確定情報として書かない）
- [ ] 誤字の機械チェック（「AI」→「AV」等）をgrepで行ったか
- [ ] 納品手順: 作業用ファイル名で作業→構文チェック→日本語ファイル名でoutputsへコピー→提示、の順を守ったか

## 過去の失敗

- 専門用語をそのまま子供向け本文に出し「少し難しい」とフィードバック → 言い換えチェックを実装後の必須ステップに
- 危険を扱う回で恐怖の着地のまま終了 → 教育効果より不安が残る
- 断定しない設計の回で、うっかり答えに寄せた記述 → 回全体の設計思想が崩れる

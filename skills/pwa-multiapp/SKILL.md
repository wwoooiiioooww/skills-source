# pwa-multiapp — 静的PWA・マルチアプリ運用の実装の型

前提とする技術構成: 単一 `index.html`（HTML/CSS/JS一体、フレームワークなし・ビルドなし）＋ PWA一式（manifest.json / sw.js / アイコン）。ホスティングは GitHub Pages。sw.jsはページ本体ネット優先方式とし、**index.html差し替え＋開き直しだけで全端末に更新が届く**（再インストール不要）状態を保つ。

## 絶対原則

1. **「組み合わせを持たせたら負け、フィルタさせたら勝ち」**
   選択肢（路線・カテゴリ等）の全組み合わせを事前生成してリスト化するのは禁止。正しい形は「単一マスタ＋実行時フィルタ関数」。マスタは1箇所、フィルタは10〜20行で済むはず。

2. **「重複排除はマスタ生成時に済ませる。実行時にチェックさせたら負け」**
   同名・表記ゆれの統合は、マスタ構築時のエイリアス（ALIAS）テーブルで1回だけ行う。

3. **「精度100%を狙わない。ざっくり許容＋事後修正」**
   AI生成の固有名詞データは「明らかな誤りは都度報告→1行修正で直せる構造」を優先する。ただし疑わしい固有名詞（実在しない可能性のある駅名等）は検索で事実確認してから断定する（「屏風浦」混入の実例あり）。

4. **「姉妹アプリ間で状態を共有させない」**
   同一ドメイン配下の複数PWAでは、SWキャッシュ名・localStorageキーの名前空間を必ずアプリ専用にする。sw.js/manifest.jsonのコピー使い回しは、片方の更新が反映されない事故を起こす。

5. **「ファイル名・配置はプラットフォーム規約に従わせる」**
   GitHub Pagesでルート自動表示されるのは `index.html` のみ（ヘルスきちの「表示されない」事故の主因）。

## データ設計の型

```js
// 単一マスタ（実体ごとに所属・属性をフラグで持つ）
const STATIONS = {
  "銀座": { name: "銀座", lines: ["G","M","H"], area: "tokyo23" },
};
// 実行時フィルタ
function getCandidates(selLines, selAreas) {
  return Object.values(STATIONS).filter(s =>
    selAreas.includes(s.area) && s.lines.some(l => selLines.includes(l)));
}
```

- 確率・表示テキストはベタ書き禁止。**重み付きテーブル（label + w）**の形で外部化し、UI側で編集可能にできる構造にする。
- 「連続で同じ結果」を防ぐ箇所はシャッフルバッグ方式等の偏り防止を検討する。
- 状態はlocalStorageの単一JSONキー（例: `appname_v1`）。**画像はIndexedDB**（localStorageは容量超過で保存自体が失敗する）。バックアップJSONに画像が含まれないことをUIに明示する。

## 実装チェックリスト

- [ ] SWキャッシュ名はアプリ固有か（例: `metro-quest-v1`）。更新時にv2, v3…とバンプしたか
- [ ] メインHTMLは `index.html` か
- [ ] localStorageキーが他アプリと衝突しない命名か
- [ ] マスタはSingle Source of Truthか。フィルタは実行時評価か
- [ ] 表記ゆれ統合はマスタ生成時のエイリアステーブルに閉じているか
- [ ] モバイル幅（実質360px級）で1行詰め込みしていないか。固定幅要素の合計を概算し、可変inputに残る幅を確認。数値inputは `min-width:0`＋中央寄せ、必要なら2段化
- [ ] 数値入力に `min` 属性＋保存時レンジ検証（負数・上限超過・正解>問題数）があるか
- [ ] ユーザー入力の表示は全箇所 `esc()` でXSS対策済みか。onclick文字列内の引用符エスケープも確認
- [ ] `<input type="time">` 等のネイティブUI部品がロケール設定（12/24時間表記）で挙動が変わらないか。曖昧なら自前実装に置換
- [ ] 子供向けアプリの「保護者のみ変更可能」項目は簡易PIN等でガードされ、デフォルトPINが明記されているか。**PINは任意設定にして必要な人だけに掛け、親にはバイパスを用意**（摩擦を全員に課さない）
- [ ] 自由入力より選択式（チップ・±ステッパー・datalist）を優先しているか。リストが多い設定は路線単位等で絞ってから選ばせる導線があるか

## 過去の失敗（再発防止）

- sw.jsコピー使い回し → キャッシュ名衝突で更新が壊れる
- 日本語ファイル名をルート表示させようとする → GitHub Pagesでは不可
- 全組み合わせの事前リスト化 → 保守不能
- 閾値ベタ書きの確率関数 → 後から調整不能。重み付きテーブルに
- ±ボタン4つ＋入力2つを1行に詰め込み → 入力欄が実質18pxに。ラベル付き2段レイアウトに
- Web Speech APIの結果を全件連結 → Android Chromeで「こんにちはこんにちは」化。resultIndexから新規分のみ処理＋final累積に末尾一致ガード＋interimは最新1件のみ
